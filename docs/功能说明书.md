## 一、这个系统是干什么的

归爻是一个**股票交易助手**。每天早上帮你做三件事：

1. **判断今天能不能做** —— 看市场大环境是安全还是危险
2. **选出今天该买什么** —— 从几百只股票里筛出最有可能涨的
3. **告诉你买多少、什么价、什么时候卖** —— 连止损止盈都算好

它的代码在**你的电脑**（开发用）和**云端服务器**（运行时）上都有一套。每天自动运行，不需要你手动操作。

## 二、系统长什么样

代码都在 `H:\guiyao_v5\` 下面。总共约 1,500 行，分 11 个文件。

### 核心模块（7 个）

| 模块 | 它干什么 | 大实话 |
|------|---------|--------|
| **数据层** | 从通达信、akshare 等免费数据源下载股票行情数据 | 相当于你的"信息员"，每天早上去菜市场看价格 |
| **市场状态检测** | 判断现在是大牛市、震荡市、还是熊市 | 就像出门前看天气预报——今天适合干活还是躲家里 |
| **因子计算** | 对每只股票算一堆指标（涨了多少、波动大不大、换手率高不高） | 相当于给每只股票打分 |
| **Numba 回测内核** | 同时模拟上百种参数组合怎么交易 | 就是"如果 A 方案，过去三年能赚多少？B 方案呢？" |
| **仓位管理** | 算出来该买多少股 | 不是全仓冲，而是算好"最多亏多少" |
| **参数搜索** | 自动找最好的交易参数 | 好比买衣服试尺码——自动帮你试几百个码数 |
| **影子账本** | 记录 AI 说了什么 vs 你做了什么 | 让你看到自己的执行力好不好 |

### 交易策略（2 个）

| 策略 | 对应账户 | 总资金 | 一句话说明 |
|------|---------|:------:|-----------|
| **ETF 轮动** | 银河证券（低手续费） | 5 万 | 每周换一次，选最强的那只 ETF 买 |
| **个股突破** | 国泰海通（万 0.8） | 5 万 | 找到放量暴涨的股票，小仓位试错 |

## 三、每天早上系统怎么工作

### 时间线（交易日）

| 时间 | 做什么 | 详细说明 |
|:----:|-------|---------|
| **9:00** | 下载数据 | 从通达信拉取 210 只自选股的日线数据，从 akshare 拉取全市场 1000 只股票的快照 |
| **9:05** | 超时检查 | 如果过了 9:25 数据还没下完，系统认为今天异常，**跳过所有买入**，只做止损检查 |
| **9:10** | 判断市场状况 | 看三点：站上均线的股票比例（宽度）、指数涨了多少（趋势）、波动大不大 |
| **9:12** | 计算选股得分 | 对 210 只股票打分（动量 + 波动率 + 反转） |
| **9:13** | 模拟试参数 | 用 48 种不同的参数组合跑回测，挑出过去最好的那个 |
| **9:14** | 算仓位 | 确定买多少股、设止损价 |
| **9:15** | 输出结果 | 告诉你今天该买什么、卖什么 |
| **15:00** | 复盘 | 对比 AI 说了什么 vs 你实际做了什么 |

### 如果出问题了

- **网络断了**：改用昨天缓存的数据，标记为"过期数据"，仓位自动打八折
- **数据卡住了**：9:25 还没跑完，直接取消今天的买入，只做止损
- **极端行情**：跌停超过 300 家，或者宽度低于 10%，**强制停止一切买入**

## 四、云端的那一套（部署在 qp-server）

云端的服务器是一个**Ubuntu 系统的微型电脑**（2 核 CPU、3.6GB 内存）。它上面运行着和你本地电脑一样的代码，但：

### 设备信息

| 项目 | 值 |
|------|-----|
| 系统 | Ubuntu 24.04 |
| CPU | 2 个虚拟核 |
| 内存 | 3.6 GB |
| 硬盘 | 够用 |
| 位置 | 腾讯云（49.232.148.142） |
| Python | 3.11（虚拟环境内） |
| SSH 登录 | `ssh qp-server`（密钥，不要密码） |

### 代码位置

```
/home/ubuntu/guiyao_v5/
├── engine/           ← 核心模块（7 个文件）
├── strategies/       ← 交易策略（2 个文件）
├── main.py           ← 每天早晨执行的入口
└── data/             ← 存放行情数据
```

### 云端装了哪些东西

- **polars**：大量数据的快速处理
- **numpy**：数字计算
- **numba**：加速计算（把 Python 代码编译成机器码）
- **mootdx**：通达信的免费数据接口
- **akshare**：另一个免费数据源

### 云端和本地的区别

| 功能 | 本地电脑 | 云端服务器 |
|------|:------:|:---------:|
| 代码开发 | ✅ 写代码、测试 | ❌ 不直接改代码 |
| 历史回测 | ✅ 跑过 384 次验证 | ❌ 不跑回测 |
| 日常运行 | ❌ 不常开 | ✅ 每天 9:00 自动跑 |
| 数据下载 | ✅ 下载过 210 只 | ✅ 直接从云端数据读 |
| 文件编辑 | ✅ 用 VSCode | ❌ 通过 SSH 上传 |

### 怎么把新代码上传到云端

```
方法一：SSH base64 管道（小文件）
  [你的电脑] → base64 编码 → SSH 连接 → [云端] 解码写入文件

方法二：tar 打包（批量文件）
  [你的电脑] → tar 压缩 → SSH 标准输入 → [云端] tar 解压

方法三：stdin 直接推送（中等文件）
  [你的电脑] → Python subprocess → 管道 → [云端] 写入
```

## 六、个股突破策略（国泰海通账户 5 万）

### 这是干什么的

每天找那种"突然放巨量上涨"的股票。这种股票背后通常有资金进场，可能继续涨。

### 选股条件

1. **股票成交额在 5 亿元以上**（大资金才进的去、出得来）
2. **最近 20 天波动最大**的 30 只（涨跌幅度大才有肉吃）
3. **当日成交量是平时 3 倍以上**（放巨量）
4. **收盘价在日内高位的 70% 以上**（不是冲高回落）

### 仓位管理（最重要！）

```
第一次买：总资金的 10%（约 5,000 元）  ← 试错
               │
               ├── 涨了 5% 以上 → 加仓到 40%（约 20,000 元）
               │                     │
               │                     └── 从最高点跌 8% → 全部卖出
               │
               └── 跌了 5% → 全部卖出（止损）
                     │
                     └── 连续两次止损 → 这个票的买入信号冻结 3 天
```

**总结**：每次只拿 5,000 块试水。对了加仓，错了砍掉。连续错两次就停手 3 天。

### 验证结果

| 指标 | 值 | 好还是差？ |
|------|:--:|:----------:|
| 盈亏比 | **4.8** | ✅ 赚一笔顶亏四笔 |
| 胜率 | **24%** | ✅ 正常（这种策略胜率就是低） |
| 测试期夏普 | **+0.53** | ✅ 良好（>0.2） |
| 最大回撤 | 18.6% | ⚠️ 略高但接受 |
| 总交易次数 | 67 笔 | ✅ 样本量够 |

## 七、影子账本——记录 AI 和你谁更厉害

系统自动在后台维护两本账：

**AI 账（影子账本）**：
- AI 发出买入信号时，假设按 15% 仓位模拟买入
- AI 发出卖出信号时，假设按规则模拟卖出
- 全程不经过你，完全自动化

**你的账（实盘账本）**：
- 你实际下单买的股票
- 你实际卖出的价格和时间

**每周日**，系统自动生成对比报告：

```
📊 影子账本 vs 实盘账本 — 周报

  AI  模拟: 5 笔 | 盈亏 +1,200
  用户实盘: 3 笔 | 盈亏 +800

  ⚠️ AI 跑赢实盘，建议下周严格执行 AI 信号。
```

**规矩**：如果连续两周实盘跑输 AI，第三周你必须无条件跟 AI 的信号。这是铁律。

## 十、紧急情况处理指南

| 情况 | 怎么做 |
|------|--------|
| 云端 SSH 连不上 | 等 5 分钟重试。两次失败后检查服务器是否欠费 |
| 策略乱发买入信号 | 直接停掉 main.py。联系作者检查 |
| 想改参数 | 改本地代码 → 本地验证 → 上传到云端 |
| 忘记今天有没有跑 | SSH 连上看看 `~/guiyao_v5/main.py` 是否运行过 |
| 需要紧急止损 | 用券商软件直接操作。AI 的建议仅供参考 |
