# 归爻 V5 — 末轮参数专家论证咨询

## 背景

归爻 V5 的核心参数已经过四轮专家论证，12 组参数完成仲裁终裁。但在最终交叉审计中发现仍有 2 组参数从未经过任何形式讨论，直接写死在代码中。它们是系统逻辑链条中最底层的两句判定——比权重更直接地影响"什么时候买"和"买什么"。

## 一、宏观脑状态阈值（BULL_THRESH=65 / BEAR_THRESH=35）

| 参数 | 当前值 | 代码位置 |
|:----|:------:|----------|
| BULL_THRESH | 65 | engine/regime.py |
| BEAR_THRESH | 35 | engine/regime.py |
| 评分范围 | 0-100分（趋势40+波动30+宽度30） | |

### 判定逻辑

```python
if total >= BULL_THRESH and (streak>=3 or ma3>=BULL_THRESH):
    return "bull"
elif total < BEAR_THRESH:
    return "bear"
else:
    return "chop"
```

Bear 状态触发一票否决（全仓货基，不买一股）。Bull 状态下全系统正常运行（ETF 正常轮动+个股正常突破）。Chop 状态下，个股突破降级为仅观察不买入，ETF 轮动继续但信号降权。

### 历史数据中的分布

以 2023-02 至 2026-07 共 827 天的宏观脑评分数据为样本（若有现成统计）：

| 当前阈值 | Bull 天数 | Chop 天数 | Bear 天数 |
|:--------:|:--------:|:---------:|:---------:|
| 65/35 | （待计算） | （待计算） | （待计算） |

如果 65/35 被证实导致 "Bull 天数 > 80%" 或 "Bear 天数 < 5%"，则阈值可能过于宽松——宏观脑失去了"止盈止损"的意义。如果 Bear 天数 > 30%，则阈值可能过严——系统大部分时间处于空仓状态，无法捕捉收益。

### 核心问题

- 65/35 的合理性依据是什么？为什么不是 70/30 或 60/40？
- 阈值和权重（40/30/30）之间存在耦合——同是总分 68 分，在 40/30/30 权重下是 Bull，如果权重改为 30/40/30 则可能变成 Chop。阈值是否需要与权重**联动调整**，还是可以独立论证？
- Bear 状态一票否决——这意味着 Bear 误判（把 Chop 判为 Bear）的代价是全仓空仓一天。Bear 误判的容忍度应该设在多高？当前阈值产生的 Bear 误判率能否通过 2015/2024 极端行情的回放验证？
- 是否需要引入"模糊边界"（例如总分为 62-68 分之间的窗口标记为 Bull-Chop 过渡区，不计入 Bull 通过率）？

## 二、个股池筛选逻辑（210 只股票构成）

| 参数 | 当前值 | 代码位置 |
|:----|:------:|----------|
| 股票数量 | 210 只 | engine/data_layer.py |
| 筛选标准 | 未记录 | 无独立文件 |
| 数据范围 | 纯 A 股主板 | engine/data_layer.py |

### 当前池子的来源（需方坦白）

初始 210 只股票的筛选逻辑来源于**一个更早的策略衍生**，具体来说：

- 最早是按某个可转债 + 双低策略的股票池衍生的
- 后来池子经过多次手动增删（业绩差的删掉、新关注的加入）
- **从来没有系统的、数据驱动的筛选标准**

目前池子中的数据（827 天日线）是完整的，但池子本身的股票构成没有任何质量控制。

### 核心问题

- 既然池子是"历史遗留产物"，是否应该进行一轮系统性的池子重建？还是直接承认池子有不完美但"有总比没有好"？
- 市场覆盖率：210 只股票能否代表全市场？如果池子集中在少数几个行业（如消费、银行），则微观脑在科技/医药行情中会天然无信号。
- 操作建议的三种路径并请给出选择理由：
  - **A：维持现有 210 只不做调整**（理由：微调不会带来可证实的收益提升，且引入新筛选标准意味着引入新的过拟合风险）
  - **B：替换为 210 只全市场流动性最好的股票**（按日均成交额排序，每月或每季度更新一次）
  - **C：不设固定个股池，改用条件过滤**（任何满足放量+振幅条件的股票都可入选，没有固定数量的限制，信号密度的控制由放量倍数负责）
